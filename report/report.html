🎯 1. Project Overview

SuperMarket Pro is a comprehensive web-based supermarket management system that streamlines retail operations. It provides complete inventory management, point of sale (POS), sales tracking, and reporting capabilities for supermarkets and retail stores.

Project Objectives

  • Create a complete supermarket management solution
  • Enable efficient inventory and stock management
  • Provide a fast and user-friendly point of sale system
  • Track sales, revenue, and generate business reports
  • Implement role-based access for different staff positions
  • Deliver a responsive interface accessible on all devices

❓ 2. Problem Statement

Traditional supermarket operations face several challenges:

  • Manual Inventory Tracking: Difficulty tracking stock levels and low stock alerts
  • Inefficient Checkout: Slow manual billing processes
  • No Centralized System: Separate systems for inventory, sales, and reporting
  • Stock Management Issues: Stockouts leading to lost sales, overstocking tying up capital
  • Lack of Real-time Analytics: No instant visibility into sales performance

Our Solution: SuperMarket Pro addresses these challenges by providing an integrated system for inventory management, point of sale, and business analytics all in one platform.

💡 3. Solution Overview

SuperMarket Pro is a complete supermarket management ecosystem with three distinct user roles:

👑 Admin

  • Full system control and oversight
  • User management (CRUD operations)
  • Product management (CRUD)
  • Category and supplier management
  • View all sales and revenue reports
  • System configuration

📊 Manager

  • Product management (add, edit, delete)
  • Inventory tracking and stock updates
  • View sales reports and analytics
  • Manage low stock alerts
  • View all transactions
  • Generate sales reports

🛒 Cashier

  • Process sales at point of sale (POS)
  • View products and prices
  • Generate invoices for customers
  • Search products by name or barcode
  • Calculate totals with tax
  • Print receipts

🗄️ 6. Database Design

The system uses 7+ tables with proper relationships and foreign keys.

Table Structure

Table Description Key Fields
users User accounts & authentication id, name, email, password, role, status
categories Product categories id, name, description, status
suppliers Product suppliers id, name, contact, phone, email
products Product inventory id, name, barcode, category_id, supplier_id, price, stock, image
sales Sales transactions id, invoice_number, user_id, total_amount, payment_method, sale_date
sale_items Individual sale items id, sale_id, product_id, quantity, unit_price, subtotal

Entity Relationship Diagram (ERD)

  • Users → Sales (one-to-many) - Cashier processes sales
  • Products → Sale Items (one-to-many) - Products in sales
  • Sales → Sale Items (one-to-many) - Sale contains items
  • Products → Categories (many-to-one) - Product categorization
  • Products → Suppliers (many-to-one) - Product sourcing

⚙️ 7. Key Functionalities by Role

Admin CRUD Operations

  • ✅ Create: Add new users, products, categories, suppliers
  • ✅ Read: View all users, products, sales, reports
  • ✅ Update: Edit user roles, product details, category information
  • ✅ Delete: Remove users, products, categories (with confirmation)
  • ✅ Activate/Deactivate: Toggle user and product status

Manager Functionalities

  • Product Management: Add, edit, and delete products with images
  • Inventory Tracking: Update stock levels, monitor low stock alerts
  • Sales Reports: View daily, weekly, monthly sales reports
  • Revenue Analytics: Track total revenue, average sale value
  • Stock Alerts: Automatic notifications for low stock items

Cashier Functionalities

  • Point of Sale (POS): Fast and intuitive checkout interface
  • Product Search: Search by name or barcode for quick billing
  • Shopping Cart: Add/remove items, adjust quantities
  • Invoice Generation: Auto-generate invoices with tax calculation
  • Payment Processing: Multiple payment methods (Cash, Card, Mobile Money)
  • Receipt Printing: Print or save receipts for customers

🔒 9. Security Features

  • Password Hashing: All passwords stored using password_hash() with BCRYPT algorithm
  • Prepared Statements: All database queries use PDO prepared statements to prevent SQL injection
  • Session Management: Secure session handling with session_regenerate_id()
  • Role-Based Access Control: Each user role has specific permissions (requireAdmin, requireManager, requireLogin)
  • XSS Protection: All user input sanitized with htmlspecialchars()
  • Form Validation: Both client-side and server-side validation
  • Status Checks: Inactive users cannot access the system
  • File Upload Security: Product images validated for type and size
